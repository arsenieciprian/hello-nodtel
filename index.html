Ce este un SIP Trunk? What is a SIP Trunk?

Un SIP Trunk este o interfață logică între domeniul privat (IP-PBX/UC) și domeniul public (PSTN/PLMN), bazată pe SIP pentru semnalizare și RTP/SRTP pentru media. Un singur link IP poate transporta simultan zeci sau sute de apeluri. A SIP trunk is the logical interface between the private domain (IP-PBX/UC) and the public domain (PSTN/PLMN), using SIP for signaling and RTP/SRTP for media. A single IP link can carry dozens or hundreds of calls concurrently.

  • Semnalizare: SIP (UDP/TCP/TLS), INVITE/ACK/BYE etc. Signaling: SIP (UDP/TCP/TLS), INVITE/ACK/BYE etc.
  • Media: RTP/SRTP, RTCP, re-INVITE (hold, codec change). Media: RTP/SRTP, RTCP, re-INVITE (hold, codec change).
  • Identitate: P-Asserted-Identity (PAI), Privacy, CLI E.164. Identity: P-Asserted-Identity (PAI), Privacy, E.164 CLI.

Arhitectură & Protocoale Architecture & Protocols

  • Securitate: TLS 1.2+, SRTP (AES_CM_128), DTLS-SRTP. Security: TLS 1.2+, SRTP (AES_CM_128), DTLS-SRTP.
  • NAT traversal: Media anchoring (SBC), symmetric RTP, rport, OPTIONS. NAT traversal: Media anchoring (SBC), symmetric RTP, rport, OPTIONS.
  • DTMF: RTP events (RFC2833/4733) recomandat. DTMF: RFC2833/4733 recommended.
  • QoS: DSCP EF (46) pentru RTP, CS3/AF31 pentru SIP. QoS: DSCP EF (46) for RTP, CS3/AF31 for SIP.
  • Redundanță: DNS SRV, terminări multiple, failover OPTIONS. Redundancy: DNS SRV, multiple terminations, OPTIONS failover.
  • Fax: T.38 sau G.711 pass-through (ptime 20). Fax: T.38 or G.711 pass-through (ptime 20).
  • Codec: G.711 A/µ-law, G.729, Opus (opțional). Codec: G.711 A/µ-law, G.729, Opus (optional).
