Ce este un SIP Trunk? [RO]

Un SIP Trunk este o interfață logică între domeniul privat (IP‑PBX/UC) și domeniul public (PSTN/PLMN), bazată pe SIP pentru semnalizare și RTP/SRTP pentru media. Un singur link IP poate transporta simultan zeci sau sute de apeluri, cu DID‑uri pentru intrare și rutare controlată pentru ieșire.

  • Semnalizare: SIP (UDP/TCP/TLS), mesaje INVITE/ACK/BYE/CANCEL/REGISTER/OPTIONS/UPDATE/PRACK/REFER.
  • Negociere media: SDP (codec, ptime, fmtp, ICE/DTLS când e cazul), early media (183 Session Progress).
  • Media: RTP/SRTP, RTCP pentru măsurarea calității; suport pentru re‑INVITE/UPDATE (hold, schimbare codec, refresh).
  • Identitate: P‑Asserted‑Identity (PAI), From/To, Privacy; prezentare CLI corectă conform E.164.

What is a SIP Trunk? [EN]

A SIP trunk is the logical interface between the private domain (IP‑PBX/UC) and the public domain (PSTN/PLMN), using SIP for signaling and RTP/SRTP for media. A single IP link can carry dozens or hundreds of calls concurrently, with inbound DIDs and controlled outbound routing.

  • Signaling: SIP (UDP/TCP/TLS), methods INVITE/ACK/BYE/CANCEL/REGISTER/OPTIONS/UPDATE/PRACK/REFER.
  • Media negotiation: SDP (codec, ptime, fmtp, ICE/DTLS when applicable), early media (183 Session Progress).
  • Media: RTP/SRTP, RTCP for quality; supports re‑INVITE/UPDATE (hold, codec change, refresh).
  • Identity: P‑Asserted‑Identity (PAI), From/To, Privacy; correct CLI presentation per E.164.

Arhitectură & Protocoale [RO]

  • Domenii: privat (IP‑PBX/UC) vs public (PSTN/PLMN); interfață prin SIP trunk protejat cu SBC.
  • Securitate: TLS 1.2+, SRTP (AES_CM_128_HMAC_SHA1_80/32), DTLS‑SRTP/SDES, certificate CA, SIPS unde e cazul.
  • NAT traversal: media anchoring, symmetric RTP, rport, keepalive (CRLF/OPTIONS), SIP ALG off.
  • Timere & refresh: SIP T1/T2, Session‑Expires (RFC4028), re‑INVITE/UPDATE, 100rel (PRACK) pentru early media.
  • DTMF: RTP events (RFC2833/4733) recomandat; alternativ SIP INFO; fallback in‑band G.711.
  • QoS: DSCP EF (46) pentru RTP, CS3/AF31 pentru SIP; jitter buffer adaptiv, MTU 1500, evitare fragmentare.
  • Redundanță: DNS SRV/A cu priorități/weight; terminări multiple; failover la pierdere OPTIONS.
  • Interconectare: IP allow‑list sau autentificare digest; rate limiting CPS; filtre destinații.
  • Fax: T.38 unde disponibil; altfel G.711 pass‑through (recomandat ptime 20, fără transcoding).
  • Codec: G.711 A/µ‑law, G.729, Opus (opțional); evitați transcodarea pe cât posibil.
  • Header‑e: P‑Asserted‑Identity, P‑Preferred‑Identity, Diversion pentru redirecționări; Privacy id.
